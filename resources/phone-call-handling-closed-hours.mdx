---
title: "zoom_phone_call_handling_closed_hours — Closed hours"
description: "Configure how a Zoom Phone extension routes calls outside of defined business hours using the zoom_phone_call_handling_closed_hours resource."
---

Closed hours call handling applies whenever an extension is outside its defined business hours schedule. You use this resource to control what happens to those calls — for example, forwarding them to voicemail, another extension, or an external number. This resource depends on `zoom_phone_call_handling_business_hours` being configured with `custom_hours.type = 2` (custom hours); closed hours routing only becomes active when the business hours schedule is set to custom rather than 24/7. Define exactly one resource per `extension_id`.

<Note>
  This resource depends on [`zoom_phone_call_handling_business_hours`](/resources/phone-call-handling-business-hours). Set `custom_hours.type = 2` on the business hours resource before applying this one, and use `depends_on` to enforce the ordering.
</Note>

<Warning>
  Some fields in `call_handling` are not returned by the Zoom API after they are written. Add a `lifecycle` block with `ignore_changes` for those fields to prevent Terraform from treating them as drift. At minimum, include `call_handling.receive_call`. See the example below for the recommended pattern.
</Warning>

## Required API permissions

This resource requires the following OAuth scopes on the server-to-server app credential used by the provider:

- `phone:read:call_handling_setting:admin`
- `phone:write:call_handling_setting:admin`
- `phone:update:call_handling_setting:admin`
- `phone:delete:call_handling_setting:admin`

## Example usage

```hcl
# Define only one resource for each extension id.
resource "zoom_phone_call_handling_closed_hours" "example" {
  extension_id = "wGJDBcnJQC6tV86BbtlXXX"

  call_handling = {
    call_not_answer_action        = 1
    busy_on_another_call_action   = 21
    connect_to_operator           = false
    allow_callers_check_voicemail = false
    voicemail_greeting_id         = "" # default
    ring_mode                     = "simultaneous"
    max_wait_time                 = 30
  }

  # some extensions support call forwarding, such as users
  call_forwarding = {
    require_press_1_before_connecting = true
    enable_zoom_mobile_apps           = true
    enable_zoom_desktop_apps          = true
    enable_zoom_phone_appliance_apps  = true
    settings = [
      {
        description  = "external person"
        enable       = true
        phone_number = "+1234567890"
      },
    ]
  }

  # closed hours routing requires business_hours custom_hours.type = 2
  depends_on = [
    zoom_phone_call_handling_business_hours.example
  ]

  lifecycle {
    ignore_changes = [
      # The Zoom API does not return this field — ignore it to prevent drift.
      call_handling.receive_call,
    ]
  }
}
```

## Schema

### Required

<ParamField path="extension_id" type="string" required>
  The unique identifier of the extension to configure. Only one `zoom_phone_call_handling_closed_hours` resource may exist per extension.
</ParamField>

<ParamField path="call_handling" type="object" required>
  Determines how calls are routed outside of business hours.

  <Note>
    Some fields in this block are not returned by the Zoom API after they are written. Use `lifecycle { ignore_changes = [call_handling.receive_call] }` to avoid perpetual drift for those fields.
  </Note>

  <Expandable title="properties">
    <ParamField path="call_not_answer_action" type="number">
      The action to take when a call is not answered:
      - `1` — Forward to a voicemail
      - `2` — Forward to the user
      - `4` — Forward to the common area
      - `6` — Forward to the auto receptionist
      - `7` — Forward to a call queue
      - `8` — Forward to a shared line group
      - `9` — Forward to an external contact
      - `10` — Forward to a phone number
      - `11` — Disconnect
      - `12` — Play a message, then disconnect
      - `13` — Forward to a message
      - `14` — Forward to an interactive voice response (IVR)
    </ParamField>

    <ParamField path="forward_to_extension_id" type="string">
      The extension ID to forward unanswered calls to. Required when `call_not_answer_action` is `2`, `4`, `6`, `7`, `8`, or `9`.
    </ParamField>

    <ParamField path="busy_on_another_call_action" type="number">
      The action to take when the extension is busy on another call:
      - `1` — Forward to a voicemail
      - `2` — Forward to the user
      - `4` — Forward to the common area
      - `6` — Forward to the auto receptionist
      - `7` — Forward to a call queue
      - `8` — Forward to a shared line group
      - `9` — Forward to an external contact
      - `10` — Forward to a phone number
      - `12` — Play a message, then disconnect
      - `21` — Call waiting
      - `22` — Play a busy signal
    </ParamField>

    <ParamField path="busy_forward_to_extension_id" type="string">
      The extension ID to forward calls to when busy. Required when `busy_on_another_call_action` is `2`, `4`, `6`, `7`, `8`, or `9`.
    </ParamField>

    <ParamField path="allow_callers_check_voicemail" type="boolean">
      Whether callers can check voicemail over the phone. Required when `call_not_answer_action` is `1` (forward to a voicemail).
    </ParamField>

    <ParamField path="ring_mode" type="string">
      The ring mode for the extension. Allowed values: `simultaneous`, `sequential`. For user closed hours, set this together with `max_wait_time`.
    </ParamField>

    <ParamField path="max_wait_time" type="number">
      The maximum number of seconds a caller waits before being rerouted. For simultaneous or sequential ring mode: `10`, `15`, `20`, `25`, `30`, `35`, `40`, `45`, `50`, `55`, `60`. For call queue overflow: additionally `120`, `180`, `240`, `300`, `600`, `900`, `1200`, `1500`, `1800`.
    </ParamField>

    <ParamField path="greeting_prompt_id" type="string">
      The ID of the greeting audio prompt. Pass an empty string for the default or `"0"` to disable. Only applies to call queue and auto receptionist extensions.
    </ParamField>

    <ParamField path="voicemail_greeting_id" type="string">
      The ID of the voicemail greeting prompt. Required when `call_not_answer_action` or `busy_on_another_call_action` is `1`. Applies to call queue, auto receptionist, and user extensions.
    </ParamField>

    <ParamField path="connect_to_operator" type="boolean">
      Whether callers can press a key to reach an operator. Required when `call_not_answer_action` or `busy_on_another_call_action` is `1` (forward to a voicemail).
    </ParamField>

    <ParamField path="operator_extension_id" type="string">
      The extension ID of the operator to transfer callers to. Required when `call_not_answer_action` is `1` and `connect_to_operator` is `true`.
    </ParamField>

    <ParamField path="phone_number" type="string">
      The external phone number in [E.164](https://en.wikipedia.org/wiki/E.164) format to forward unanswered calls to. Required when `call_not_answer_action` is `10`.
    </ParamField>

    <ParamField path="phone_number_description" type="string">
      An optional description for the external forwarding number. Used when `call_not_answer_action` is `10`.
    </ParamField>

    <ParamField path="busy_phone_number" type="string">
      The external phone number in [E.164](https://en.wikipedia.org/wiki/E.164) format to forward busy calls to. Required when `busy_on_another_call_action` is `10`.
    </ParamField>

    <ParamField path="busy_phone_number_description" type="string">
      An optional description for the busy forwarding number. Used when `busy_on_another_call_action` is `10`.
    </ParamField>

    <ParamField path="overflow_play_callee_voicemail_greeting" type="boolean">
      Whether to play the callee's voicemail greeting when the caller reaches the end of the unanswered forwarding sequence. Applies when `call_not_answer_action` is `2`, `4`, `6`, `7`, `8`, `9`, or `10`.
    </ParamField>

    <ParamField path="play_callee_voicemail_greeting" type="boolean">
      Whether to play the callee's voicemail greeting when `busy_on_another_call_action` or `call_not_answer_action` is `1` (forward to a voicemail).
    </ParamField>

    <ParamField path="busy_play_callee_voicemail_greeting" type="boolean">
      Whether to play the callee's voicemail greeting when the caller reaches the end of the busy forwarding sequence. Applies when `busy_on_another_call_action` is `2`, `4`, `6`, `7`, `8`, `9`, or `10`.
    </ParamField>

    <ParamField path="unanswered_require_press1_before_connecting" type="boolean">
      When a call is unanswered and forwarded to an external contact or number, whether the recipient must press `1` before the call connects. Prevents missed calls from reaching external voicemail.
    </ParamField>

    <ParamField path="busy_require_press1_before_connecting" type="boolean">
      When the extension is busy and the call is forwarded to an external contact or number, whether the recipient must press `1` before the call connects.
    </ParamField>
  </Expandable>
</ParamField>

### Optional

<ParamField path="call_forwarding" type="object">
  Configures forwarding of calls to personal phone numbers or Zoom apps. Supported on user extensions.

  <Expandable title="properties">
    <ParamField path="require_press_1_before_connecting" type="boolean">
      Whether the recipient must press `1` before a forwarded call connects to a personal phone number. Prevents missed calls from reaching personal voicemail. Always enabled and required for call queue extensions.
    </ParamField>

    <ParamField path="enable_zoom_mobile_apps" type="boolean">
      Whether to forward calls to Zoom Mobile Apps.
    </ParamField>

    <ParamField path="enable_zoom_desktop_apps" type="boolean">
      Whether to forward calls to Zoom Desktop Apps.
    </ParamField>

    <ParamField path="enable_zoom_phone_appliance_apps" type="boolean">
      Whether to forward calls to Zoom Phone Appliance Apps.
    </ParamField>

    <ParamField path="settings" type="set of objects">
      External phone numbers to forward calls to.

      <Expandable title="properties">
        <ParamField path="phone_number" type="string">
          The external phone number in [E.164](https://en.wikipedia.org/wiki/E.164) format.
        </ParamField>

        <ParamField path="description" type="string">
          A description for the external phone number.
        </ParamField>

        <ParamField path="enable" type="boolean">
          Whether this forwarding number is active and will receive calls.
        </ParamField>

        <ResponseField name="id" type="string">

        </ResponseField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

## Import

Import an existing closed hours configuration using the extension ID:

```bash
# ${extension_id}
terraform import zoom_phone_call_handling_closed_hours.example t6wyhAZRQXXX_Rv3jj3XXX
```
