---
title: "zoom_phone_call_queue — Route calls to user groups"
description: "Use the zoom_phone_call_queue resource to create and manage Zoom Phone call queues that route incoming calls to groups of users or common areas."
---

Call queues distribute incoming calls across a group of users, letting you handle volume for departments like sales, support, or billing without calls going to a single person. Each queue gets its own extension number and can be assigned phone numbers, members, and policies through separate resources. Defining call queues in Terraform lets you version-control your call routing configuration and replicate it consistently across environments.

## Required API permissions

This resource requires the following OAuth scopes on the server-to-server app credential used by the provider:

- `phone:read:call_queue:admin`
- `phone:write:call_queue:admin`
- `phone:update:call_queue:admin`
- `phone:delete:call_queue:admin`

## Example usage

Basic call queue:

```hcl
resource "zoom_phone_call_queue" "example" {
  name             = "terraform-example"
  extension_number = "123"
}
```

Call queue with all optional attributes, set to inactive:

```hcl
resource "zoom_phone_call_queue" "inactive" {
  name             = "terraform-example-inactive"
  extension_number = "124"
  description      = "Example description"
  cost_center      = "XXX"
  department       = "YYY"
  status           = "inactive"
}
```

## Schema

### Required

<ParamField path="extension_number" type="number" required>
  The extension number for the call queue. Must be unique within the account or site.
</ParamField>

<ParamField path="name" type="string" required>
  The display name of the call queue.
</ParamField>

### Optional

<ParamField path="cost_center" type="string">
  The cost center name to associate with this call queue. Used for internal accounting and reporting.
</ParamField>

<ParamField path="department" type="string">
  The department name to associate with this call queue.
</ParamField>

<ParamField path="description" type="string">
  A human-readable description of the call queue's purpose.
</ParamField>

<ParamField path="site_id" type="string">
  The unique identifier of the site to assign this call queue to. Required when [multiple sites](https://support.zoom.us/hc/en-us/articles/360020809672-Managing-Multiple-Sites) are enabled on the account. Retrieve site IDs from the [List Phone Sites](https://marketplace.zoom.us/docs/api-reference/phone/methods#operation/listPhoneSites) API or from the `zoom_phone_site` resource.
</ParamField>

<ParamField path="status" type="string">
  The status of the call queue. Allowed values: `active`, `inactive`. Defaults to `active` when not specified.
</ParamField>

### Read-only

<ResponseField name="id" type="string">
  The unique identifier of the call queue, assigned by Zoom. Use this value as `call_queue_id` in the `zoom_phone_call_queue_members`, `zoom_phone_call_queue_phone_numbers`, and `zoom_phone_call_queue_policy_voice_mail` resources.
</ResponseField>

<ResponseField name="extension_id" type="string">
  The extension ID assigned by Zoom.
</ResponseField>

## Import

Import an existing call queue using its call queue ID:

```bash
# ${call_queue_id}
terraform import zoom_phone_call_queue.example wGJDBcnJQC6tV86BbtlXXX
```
