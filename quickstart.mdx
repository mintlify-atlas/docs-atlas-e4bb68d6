---
title: "Get started with the Zoom Terraform provider"
description: "Install the folio-sec/zoom Terraform provider, configure Server-to-Server OAuth credentials, and create your first Zoom Phone resource."
---

This guide walks you through adding the Zoom Terraform provider to a new or existing Terraform project and provisioning a Zoom Phone resource. By the end you will have a working configuration you can extend with the full set of resources the provider supports.

<Steps>
  <Step title="Prerequisites">
    Before you begin, make sure you have the following:

    - **Terraform** ≥ 1.0 installed. Download from [developer.hashicorp.com/terraform/downloads](https://developer.hashicorp.com/terraform/downloads).
    - A **Zoom account** with Zoom Phone enabled. Zoom Phone must be active on your account for the phone resources to be available.
    - Access to the [Zoom Marketplace](https://marketplace.zoom.us/user/build) to create a Server-to-Server OAuth app.
  </Step>

  <Step title="Create a Zoom Server-to-Server OAuth app">
    The provider authenticates using Zoom's Server-to-Server OAuth flow. You need to create a dedicated app in the Zoom Marketplace to obtain credentials.

    1. Go to [marketplace.zoom.us/user/build](https://marketplace.zoom.us/user/build) and sign in.
    2. Click **Develop** → **Build App** and choose **Server-to-Server OAuth**.
    3. Give the app a name, for example `terraform-provider`.
    4. Open the **App Credentials** tab and copy your **Account ID**, **Client ID**, and **Client Secret**.
    5. Under **Scopes**, add the phone scopes required by the resources you plan to manage, such as `phone:read:admin` and `phone:write:admin`.
    6. Click **Activate** to publish the app inside your account.

    <Note>
      Server-to-Server OAuth apps are account-level and do not require user authorization. They are the recommended authentication method for machine-to-machine integrations like Terraform.
    </Note>
  </Step>

  <Step title="Configure the provider">
    Create a `main.tf` file in your Terraform working directory with the following provider block. Update the `version` constraint to pin the release you want to use.

    ```hcl main.tf
    terraform {
      required_providers {
        zoom = {
          source  = "folio-sec/zoom"
          version = "~> 0.0.0"
        }
      }
    }

    provider "zoom" {
      account_id    = var.zoom_account_id
      client_id     = var.zoom_client_id
      client_secret = var.zoom_client_secret
    }

    variable "zoom_account_id" {}

    variable "zoom_client_id" {}

    variable "zoom_client_secret" {
      sensitive = true
    }
    ```

    The provider also reads credentials from the `ZOOM_ACCOUNT_ID`, `ZOOM_CLIENT_ID`, and `ZOOM_CLIENT_SECRET` environment variables when the corresponding argument is omitted from the `provider` block. See [Authentication](/authentication) for both approaches.

    Run `terraform init` to download the provider plugin:

    ```bash
    terraform init
    ```
  </Step>

  <Step title="Create your first resource">
    Add a `zoom_phone_site` resource to declare a Zoom Phone site—a physical or logical location that groups phone users:

    ```hcl main.tf
    resource "zoom_phone_site" "headquarters" {
      name = "headquarters"

      main_auto_receptionist = {
        name = "HQ Auto Receptionist"
      }

      default_emergency_address = {
        address_line1 = "123 Main St"
        address_line2 = "Suite 100"
        city          = "San Jose"
        country       = "US"
        state_code    = "CA"
        zip           = "95131"
      }
    }
    ```

    Or create a call queue with `zoom_phone_call_queue`:

    ```hcl main.tf
    resource "zoom_phone_call_queue" "support" {
      name             = "support"
      extension_number = "200"
      description      = "Customer support queue"
    }
    ```
  </Step>

  <Step title="Run terraform plan and apply">
    Preview the changes Terraform will make:

    ```bash
    terraform plan \
      -var="zoom_account_id=$ZOOM_ACCOUNT_ID" \
      -var="zoom_client_id=$ZOOM_CLIENT_ID" \
      -var="zoom_client_secret=$ZOOM_CLIENT_SECRET"
    ```

    If the plan looks correct, apply it:

    ```bash
    terraform apply \
      -var="zoom_account_id=$ZOOM_ACCOUNT_ID" \
      -var="zoom_client_id=$ZOOM_CLIENT_ID" \
      -var="zoom_client_secret=$ZOOM_CLIENT_SECRET"
    ```

    <Tip>
      Set `ZOOM_ACCOUNT_ID`, `ZOOM_CLIENT_ID`, and `ZOOM_CLIENT_SECRET` as environment variables and omit the provider arguments from HCL to avoid passing `-var` flags on every command.
    </Tip>

    Terraform calls the Zoom API to create the resources and records state. Subsequent runs only modify resources that differ from the declared configuration.
  </Step>
</Steps>
